# KittenTTS WebUI 前端交互测试手册

## 测试环境准备

### 1. 安装依赖
```bash
pip install -r requirements.txt
```

### 2. 启动WebUI服务
```bash
python run_webui.py
```

### 3. 访问WebUI
打开浏览器访问: `http://localhost:5000`

---

## 测试套件一: 基础界面测试

### 测试1.1: 页面加载验证
**测试目的**: 验证WebUI页面能正常加载

**测试步骤**:
1. 打开浏览器访问 `http://localhost:5000`
2. 等待页面加载完成

**验证要点**:
- [ ] 页面标题显示 "KittenTTS WebUI"
- [ ] 页面头部显示 "🐱 KittenTTS WebUI"
- [ ] 左侧面板显示 "🎙️ 实时合成"
- [ ] 右侧面板显示 "任务队列" 和 "历史记录" 标签页
- [ ] 页面底部没有JavaScript错误（检查浏览器控制台）

**预期结果**: 页面完全加载，无明显错误

---

### 测试1.2: 主题切换功能
**测试目的**: 验证明暗模式切换功能

**测试步骤**:
1. 找到页面右上角的主题切换按钮（初始显示 🌙）
2. 点击按钮切换到深色模式
3. 再次点击切换回浅色模式
4. 刷新页面，检查主题是否保持

**验证要点**:
- [ ] 点击 🌙 按钮后，按钮变为 ☀️
- [ ] 页面背景变为深色渐变
- [ ] 卡片背景变为深色
- [ ] 文字颜色变为浅色
- [ ] 再次点击后恢复浅色模式
- [ ] 刷新页面后主题保持上次选择

**预期结果**: 主题切换功能正常，设置持久化

---

## 测试套件二: 文本输入与合成测试

### 测试2.1: 英文文本合成
**测试目的**: 验证英文文本合成功能

**测试步骤**:
1. 在文本输入框中输入英文文本: `Hello, this is a test of KittenTTS.`
2. 选择模型: "Mini (80M) - 最高质量"
3. 选择语音: "Bruno"（男声）
4. 语速设置为 1.0x
5. 勾选 "文本预处理" 选项
6. 点击 "生成语音" 按钮

**验证要点**:
- [ ] 点击后按钮变为 "生成中..." 并显示加载动画
- [ ] 进度条显示并逐渐推进
- [ ] 状态提示显示: "正在加载模型..." → "正在处理文本..." → "正在生成语音..."
- [ ] 生成完成后进度条达到100%
- [ ] 音频播放器显示并可播放
- [ ] 字幕显示区域显示原文
- [ ] 语言标签显示 "英文"

**预期结果**: 音频成功生成，可正常播放

---

### 测试2.4: 语速调节测试
**测试目的**: 验证语速调节功能

**测试步骤**:
1. 输入文本: `这是语速测试。`
2. 分别测试以下语速:

   - 1.0x (正常)

**验证要点**:
- [ ] 滑动语速滑块时，右侧数值实时更新
- [ ] 0.5x生成的音频播放时间明显更长
- [ ] 2.0x生成的音频播放时间明显更短
- [ ] 各语速下音频质量正常

**预期结果**: 语速调节功能正常，播放时间符合预期

## 测试套件三: 文件上传功能测试

### 测试3.1: TXT文件上传
**测试目的**: 验证TXT文件上传解析功能

**测试步骤**:
1. 创建一个测试文本文件 `test.txt`:

   ```
   Hello from file upload.
   KittenTTS is great!
   ```
2. 点击文件上传区域或拖拽文件到上传区域
3. 选择创建的 `test.txt` 文件

**验证要点**:
- [ ] 鼠标悬停在上传区域时，边框变色
- [ ] 拖拽文件到上传区域时，显示高亮
- [ ] 选择文件后，文件信息显示区域显示:
  - "已加载文件: test.txt (XX 字节)"
- [ ] 文本输入框自动填充文件内容
- [ ] 显示成功提示: "文件加载成功！文本已填入输入框"

**预期结果**: 文件上传成功，内容正确解析

## 附录: 自动化测试脚本说明

### 脚本列表
1. `test_backend.py` - 后端推理服务单元测试
   - 中文处理器测试
   - 英文文本预处理测试
   - 文本清理器测试
   - 模型加载测试（可选）

2. `test_webui_api.py` - WebUI API集成测试
   - 基础API测试（首页、模型列表、语音列表）
   - 异常场景测试
   - 完整工作流测试

3. `test_error_cases.py` - 异常场景专项测试
   - 空文本处理
   - 特殊字符处理
   - 超长文本处理
   - 文件上传错误
   - 参数验证

4. `run_all_tests.py` - 综合测试入口
   - 依赖检查
   - 项目结构检查
   - 运行所有测试套件
   - 生成测试报告

### 运行方式
```bash
# 运行单个测试套件
python test_backend.py
python test_webui_api.py
python test_error_cases.py

# 运行所有测试
python run_all_tests.py
```
